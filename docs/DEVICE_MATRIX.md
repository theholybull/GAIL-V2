# Device Matrix

## uConsole

- default mode: Work Mode
- visuals: face or bust with blurred work background
- strengths: quick capture, lists, reminders, tasks, parts, music, practical control

## Kiosk

- default mode: Home/Shop Mode
- visuals: richer body, room themes, more animation density
- strengths: presence, project support, memory-rich conversation, dashboards

## iPhone

- default mode: ask or infer
- visuals: bust or simplified 3D
- strengths: mobile chat, camera tasks later, approvals, memory access

## Watch

- default mode: Lightweight Mode
- visuals: face-only
- strengths: status, prompts, approvals, presence token
- current backend policy: watch devices can access health and RAM-only private-session endpoints, but persistent organizer routes are blocked
- approval-sensitive flows now also require the specific device ID to be registered and trusted
